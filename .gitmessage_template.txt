# [플랫폼] 타입: 간결한 제목 (50자 이내)
#
# 상세 설명 (선택사항, 필요시 작성)
# - 무엇을 변경했는지
# - 왜 변경했는지
# - 어떤 영향이 있는지
#
# 관련 이슈 (이슈 번호 입력)
# Closes #
# Fixes #
# Refs #
#
# =============== 작성 가이드 ===============
# 플랫폼:
#   [BE]      - 백엔드 API
#   [FE]      - 프론트엔드 웹
#   [iOS]     - iOS 앱
#   [Android] - Android 앱
#   [All]     - 전체 플랫폼
#
# 타입:
#   feat      - 새 기능 추가
#   fix       - 버그 수정
#   docs      - 문서 변경
#   style     - 코드 스타일 변경
#   refactor  - 코드 리팩토링
#   test      - 테스트 추가/수정
#   chore     - 빌드/설정 변경
#
# 제목 작성 규칙:
#   - 50자 이내로 간결하게
#   - 명령문으로 작성 ("추가" O, "추가했다" X)
#   - 첫 글자 소문자
#   - 마침표 없음
#
# 이슈 연결:
#   - Closes #123: 이슈 자동 완료
#   - Fixes #123: 버그 수정시 사용
#   - Refs #123: 이슈 참조만
#
# 예시:
#   [BE] feat: OAuth 로그인 API 구현
#   [iOS] fix: 앱 크래시 버그 수정
#   [Android] style: 로그인 화면 레이아웃 개선
# ==========================================
